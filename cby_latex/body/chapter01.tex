\chapter{绪论}

\section{研究背景与意义}

蛋白质是生命活动的主要承担者，在信号传导、物质运输、免疫防御、基因表达调控等几乎所有生物学过程中发挥着核心作用\cite{eisenberg2000protein}。要深入理解生物系统在分子尺度上的行为机制，就必须阐明蛋白质的功能。蛋白质功能预测不仅对基础生物学研究具有重要的科学价值，也在药物靶点发现、酶工程、合成生物学等应用领域具有广阔的前景。

国际生物信息学界通常采用基因本体（Gene Ontology, GO）系统\cite{ashburner2000gene}对蛋白质功能进行标准化描述。GO系统将蛋白质功能划分为三大本体论：生物过程（Biological Process, BP）、分子功能（Molecular Function, MF）和细胞组分（Cellular Component, CC），并以有向无环图（Directed Acyclic Graph, DAG）的形式组织数千个功能术语之间的层级关系\cite{goeman2008multiple}。在这种结构中，子节点功能是父节点功能的具体化，使得功能描述既具有高度的规范性，也蕴含着丰富的语义层级信息。

然而，随着高通量测序技术的飞速发展，蛋白质序列数据呈指数级增长，序列数据与功能注释之间的鸿沟日益加深。截至目前，UniProt/Swiss-Prot数据库\cite{uniprot2025uniprot}中收录的蛋白质中，经过实验验证获得功能注释的比例不到1\%。这一巨大的缺口促使研究者开发自动化蛋白质功能预测（Automated Function Prediction, AFP）方法\cite{friedberg2006automated}，以计算手段大规模地为未注释蛋白质分配GO功能标签。

从方法论角度看，自动功能预测方法已经从单一模态分析逐步过渡到复杂的多源信息整合\cite{lin2024comprehensive}。传统的基于序列的方法依赖同源性转移或局部基序匹配，如Diamond\cite{diamond}等比对工具通过序列相似性在已知蛋白质中搜索同源物来传递功能注释。基于网络的方法利用蛋白质-蛋白质相互作用（PPI）网络根据生物学上下文推断功能。基于结构的方法则利用三维折叠信息捕获保守的功能特征。然而，正如近期的综合综述所指出的\cite{lin2024comprehensive}，依赖单一信息来源往往限制了预测覆盖范围，尤其是对于序列相似性低或互作组数据不完整的蛋白质。

为解决上述问题，深度学习和混合信息范式已成为主流方法。早期的深度学习模型如DeepGOCNN\cite{kulmanov2020deepgoplus}专注于通过卷积神经网络捕获序列基序。为进一步增强表示能力，研究者开始将多种生物特征整合到统一的神经网络框架中。例如，Struct2GO\cite{jiao2023struct2go}通过图神经网络融入三维结构特征，DPFunc\cite{wang2025dpfunc}则将蛋白质域信息与进化序列互补融合以捕获模块化的功能模式。

近期研究进一步转向整合丰富的语义知识和多模态数据以提升泛化能力。DeepGO-SE\cite{kulmanov2024protein}引入了神经符号方法，将蛋白质语言模型与GO本体公理形式化推理相结合。ProtGO\cite{wang2025protgo}建立了融合四种模态的框架。ProtNote\cite{char2025protnote}开创性地将大语言模型（LLM）应用于蛋白质功能注释，将氨基酸序列与文本化的功能描述直接对齐，首次实现了真正意义上的零样本功能预测。

尽管取得了显著进步，现有方法仍面临两大核心挑战：

第一，\textbf{模态融合不充分}。多数方法仅利用蛋白质序列或结构的单一模态，或者仅将多种模态进行简单拼接，未能充分挖掘蛋白质序列进化信号、蛋白质域功能描述、GO标签语义定义等不同模态之间的深层互补关系。蛋白质域（domain）是蛋白质中独立折叠和行使功能的结构单元，InterPro数据库\cite{blum2025interpro}中超过80\%的蛋白质具有域注释信息，其文本描述蕴含着丰富的功能线索，但现有方法大多将其视为简单的分类标签，忽略了其文本语义。此外，ProtNote虽然引入了大语言模型，但仍依赖简单的one-hot编码表示蛋白质序列，忽略了蛋白质语言模型所能提供的丰富进化特征，且忽视了蛋白质域等关键模态，导致其在有监督场景下性能不佳。

第二，\textbf{零样本泛化能力不足}。传统的功能预测方法将每个GO标签视为独立的分类标识符（即one-hot编码），模型的预测能力被限制在训练集中出现过的标签范围内。然而，GO本体处于持续更新之中，每年都有新的功能术语被添加。对于训练集中从未出现的新功能标签，传统模型无法给出有效预测。虽然DeepGOZero\cite{kulmanov2022deepgozero}尝试利用GO本体的逻辑公理实现零样本预测，但其效果受限于本体公理的覆盖范围。现有方法多采用简单的特征拼接策略，无法根据模态质量动态调整权重，容易引入噪声，缺乏有效的机制来过滤冗余信息并增强模型在面对不完整数据时的鲁棒性。

针对上述两大挑战，本文的核心思路是：利用大语言模型（Large Language Model, LLM）对蛋白质域描述和GO功能定义进行统一的语义编码，将蛋白质功能预测从传统的封闭标签集多标签分类问题，重新定义为开放语义空间中的"蛋白质-功能"匹配问题。这一范式转换使得模型能够通过文本语义相似性推断未见过的功能标签，从根本上赋予模型零样本推理的能力。在此基础上，本文围绕多模态语义融合与预训练域感知适配两个方向展开研究，提出了两个递进式的蛋白质功能预测框架，旨在推进蛋白质功能注释的自动化水平。

\section{国内外研究现状}

蛋白质功能预测是生物信息学领域的经典问题，CAFA（Critical Assessment of Function Annotation）\cite{zhou2019cafa}挑战赛是该领域最具影响力的国际评测平台，推动了大量方法的发展。本节从四个方面系统梳理国内外研究现状。

\subsection{基于蛋白质序列的功能预测方法}

基于蛋白质序列的方法是功能预测的基础范式。早期工作主要依赖序列比对技术，遵循"序列相似则功能相似"的假设。BLAST\cite{altschul1990basic}通过局部序列比对在已知蛋白质数据库中搜索同源序列，将同源蛋白质的功能注释传递给查询蛋白质。Diamond\cite{diamond}在保持比对灵敏度的同时大幅提升了搜索速度，适用于大规模蛋白质数据库的功能注释。然而，这类方法的根本局限在于：当查询蛋白质缺乏近缘同源序列时，预测精度将急剧下降。

深度学习的兴起为序列特征的自动提取提供了强大的工具。DeepGOPlus\cite{kulmanov2020deepgoplus}采用卷积神经网络（CNN）\cite{o2015introduction}直接从蛋白质氨基酸序列中捕获局部序列基序，结合序列比对分数实现功能预测。蛋白质语言模型（Protein Language Model, PLM）的出现更是带来了范式性的变革。ESM-2\cite{lin2023evolutionary}是一种基于Transformer\cite{vaswani2017attention}架构的大规模蛋白质语言模型，在数百万条蛋白质序列上进行自监督预训练，能够提取蕴含进化信息和理化性质的高质量序列嵌入表示。以ESM-2为代表的蛋白质语言模型已成为蛋白质功能预测领域的标准特征提取器。

\subsection{基于蛋白质结构的功能预测方法}

蛋白质的三维结构是理解其功能的关键。基于"结构决定功能"的原则，利用蛋白质结构信息可以显著提升功能预测的准确性。AlphaFold2\cite{jumper2021highly}在蛋白质结构预测领域取得了革命性的突破，使得大规模获取蛋白质三维结构成为可能，极大地推动了基于结构的功能预测方法的发展。

DeepFRI利用图卷积网络（GCN）在蛋白质残基接触图上进行消息传递，提取结构拓扑特征用于功能预测。Struct2GO\cite{jiao2023struct2go}进一步提出了基于图池化算法的框架，结合AlphaFold2预测的结构信息，通过多层次的图表征学习实现了更精确的功能注释。这些方法证明了结构信息在功能预测中的重要价值，但其性能受限于结构预测的准确性，且难以对缺乏可靠结构信息的蛋白质进行有效预测。

\subsection{基于多模态信息融合的功能预测方法}

为克服单一模态的局限性，研究者开始探索融合序列、结构、文本等多种信息来源的多模态方法。

DeepGO-SE\cite{kulmanov2024protein}提出了一种神经符号方法，将蛋白质语言模型与基于GO本体公理的形式化推理相结合，以近似语义蕴涵的方式进行功能预测。PO2GO\cite{li2024partial}利用偏序嵌入方法PO2Vec捕获GO有向无环图中的拓扑关系，增强了标签空间的表示能力。ProtGO\cite{wang2025protgo}建立了一个融合四种模态的框架，包括PLM序列表示、结构特征、GO本体嵌入和可解释的文本描述。DPFunc\cite{wang2025dpfunc}将蛋白质域信息与序列数据互补融合，利用域作为功能基序来增强预测精度。

在大语言模型（LLM）驱动的方向上，ProtNote\cite{char2025protnote}开创性地将LLM应用于蛋白质功能注释，将氨基酸序列与文本化的功能描述进行直接对齐。然而，ProtNote依赖简单的one-hot编码表示序列信息，忽略了蛋白质语言模型能够提供的丰富进化特征，导致其在有监督场景下性能不佳。此外，ProtNote对所有三个本体联合训练的策略限制了模型学习各本体特有特征的能力。

\subsection{蛋白质表示学习中的预训练方法}

预训练技术在蛋白质表示学习中发挥着越来越重要的作用。蛋白质语言模型如ESM-2\cite{lin2023evolutionary}通过在大规模蛋白质序列语料上进行掩码语言建模预训练，学习到的表示蕴含了丰富的进化和功能信息。然而，这些通用预训练表示未必能充分捕获特定下游任务所需的信息。

参数高效微调（Parameter-Efficient Fine-Tuning, PEFT）技术为解决这一问题提供了优雅的方案。适配器（Adapter）\cite{houlsby2019adapter}是一类轻量级的可学习模块，以即插即用的方式嵌入预训练模型的Transformer层中，在保持原始参数冻结的同时学习任务特定的适配信息。这种策略不仅避免了全参数微调的高昂计算开销，还能有效防止灾难性遗忘。

对比学习\cite{chen2020simple}作为一种自监督学习范式，通过拉近正样本对的表示、推远负样本对的表示来学习有意义的特征表示。InfoNCE损失\cite{oord2018representation}是对比学习中最常用的目标函数。在蛋白质领域，S-PLM\cite{wang2025splm}开创性地将适配器技术与对比学习相结合，通过在ESM2的Transformer层中注入适配器模块，利用序列-结构对比学习将蛋白质三维结构信息蒸馏至序列编码器中，使其仅凭序列输入即可生成包含结构感知的表示。S-PLM的成功证明了"通过对比学习将跨模态知识蒸馏至适配器"这一技术路径的有效性，为本文第四章提出的MZSGO-DA方法提供了重要的灵感来源——将蒸馏目标从结构信息拓展至蛋白质域的功能语义信息，从而增强序列编码器的功能感知能力。

\subsection{现有研究的不足与本文的出发点}

综合上述研究现状，本文识别出现有方法的主要不足：

（1）\textbf{语义信息利用不充分}：大多数方法将GO功能标签和蛋白质域信息视为离散的分类标识符，忽略了其文本定义中蕴含的丰富语义信息。这种做法不仅限制了多模态融合的深度，更从根本上阻碍了模型对未见标签的推理能力。

（2）\textbf{多模态融合机制粗糙}：现有多模态方法大多采用简单拼接的方式整合不同模态的特征，缺乏动态适应不同样本特点的融合机制，难以有效应对不同模态信息质量参差不齐的现实情况。

（3）\textbf{零样本场景表现不佳}：受限于封闭标签集的分类范式，大多数方法无法预测训练集中未出现过的新功能标签。即使是声称支持零样本的方法，如DeepGOZero和ProtNote，也存在预测精度低或有监督性能受损等问题。

（4）\textbf{序列编码器缺乏功能先验}：通用的蛋白质语言模型通过序列自监督预训练获得表示，但这些表示不一定包含特定于功能预测任务的知识，如蛋白质域的功能信息。

本文正是针对上述不足展开研究，力图通过多模态语义融合与预训练域感知适配两条技术路径，系统性地提升蛋白质功能预测在有监督和零样本两个场景下的性能。

\section{论文主要工作与创新点}

本文围绕蛋白质功能预测这一核心任务，提出了两个递进式的方法框架。论文的主要工作和创新点如下：

（1）\textbf{提出多模态零样本蛋白质功能注释方法MZSGO}。将蛋白质功能预测重新定义为"蛋白质-GO"匹配问题，利用蛋白质语言模型ESM2提取序列进化特征，利用大语言模型Qwen3-Embedding对蛋白质域描述和GO功能定义进行统一语义编码。设计了自适应门控融合模块，根据各模态特征的共享上下文动态分配融合权重，克服了传统简单拼接方式的局限；同时引入非对称特征丢弃策略，在训练阶段随机遮蔽蛋白质侧特征以增强模型鲁棒性。MZSGO在三个GO本体的有监督与零样本评测中均达到了最优或接近最优的水平，尤其在零样本指标上大幅超越现有方法。

（2）\textbf{提出基于域感知适配器预训练的端到端蛋白质功能预测方法MZSGO-DA}。在MZSGO的基础上，进一步针对序列编码器缺乏功能先验的问题，设计了轻量级适配器模块，以即插即用方式注入蛋白质语言模型的Transformer层内部（注意力层后和前馈网络层后），通过InfoNCE对比学习预训练将蛋白质域的语义信息蒸馏至序列编码器。在下游阶段，冻结预训练适配器并引入第二组可训练适配器，以双适配器策略实现端到端训练，有效兼顾了预训练知识保留与任务特定适配。MZSGO-DA通过端到端训练避免了静态嵌入提取的信息损失，进一步提升了功能预测性能。

\section{论文组织结构}

本文共分为四章，各章内容安排如下：

第一章为绪论。介绍蛋白质功能预测的研究背景与意义，系统梳理国内外研究现状，分析现有方法的不足，阐述本文的主要工作与创新点。

第二章为相关工作与技术基础。系统介绍蛋白质功能预测相关的基础概念和核心技术，包括基因本体系统、蛋白质语言模型、大语言模型文本嵌入、多标签分类、Transformer架构与注意力机制、对比学习等，为后续章节奠定理论基础。

第三章为多模态零样本蛋白质功能注释方法MZSGO。详细介绍MZSGO的方法设计，包括多模态特征提取、特征投影与非对称丢弃、自适应门控融合机制和优化目标。在标准测试集和纯零样本测试集上与十种基线方法进行全面对比实验，并通过消融实验和参数敏感性分析验证各模块的有效性。

第四章为基于域感知适配器预训练的端到端蛋白质功能预测方法MZSGO-DA。受S-PLM利用对比学习蒸馏结构信息至适配器的启发，本章将蒸馏目标拓展至蛋白质域的功能语义信息。介绍适配器注入机制、对比学习预训练策略和双适配器端到端下游训练流程。通过实验验证端到端训练相对于静态嵌入方法的优势，并分析不同预训练策略对下游性能的影响。

最后为总结与展望，对全文工作进行总结，分析现有方法的局限性，并展望未来的研究方向。
